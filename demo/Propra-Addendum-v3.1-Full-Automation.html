This document adds everything missing from v3.0: the complete zero-touch automation model, planning permission workflow, legal & conveyancing, procurement, marketing brochure generation, mapping & satellite, rendering visuals, transaction room, and explicit definitions of what the system does automatically with no human involvement at any step.



The most important sentence in this document

At no point in any Propra workflow does a human at Propra do anything on behalf of the user. Every quote is machine-generated from a live API. Every switch is executed by Propra software calling a supplier API. Every document is generated by AI. Every notification is automated. The word "we called" or "we contacted" never appears anywhere in Propra. Propra is software. It does not make phone calls. It does not send emails on behalf of the user asking for quotes. It calls APIs and returns structured data.

C Planning Permission — Full Workflow

From automatic assessment to rendered visuals to submitted application

What this is NOT

A page that shows planning application status from the portal. That is a monitoring tool. This is a full planning workflow that assesses what is possible, generates the analysis, creates visual renderings of proposed schemes, and submits applications — all via software.



C.1 Automatic planning assessment on property add

  • Propra fetches current planning history and decisions from Planning Portal API and local authority feeds

  • AI analyses the planning history, current use class, and surrounding context

  • Permitted development rights assessed against current GPDO (England) or equivalent — what the owner can do without full planning permission

  • Outputs: PD rights summary, any prior approval requirements, full planning needed/not needed

  • All of this happens automatically when an address is added — no user request needed



C.2 Planning appraisal report — AI generated

  • Owner selects a proposed use or change they are considering (e.g. "Convert ground floor from E class to residential")

  • Propra AI generates a full planning appraisal: policy context, local plan policies, recent decisions on comparable applications, officer report tone analysis, likely outcome assessment

  • Report is branded PDF, ready to share with planning consultant or use as briefing document

  • Comparable decisions: Propra pulls recent planning decisions for similar applications in the same LPA to calibrate success probability

  • This does not replace a qualified planning consultant for complex applications — but it replaces the initial £3k-£5k feasibility report

E Procurement & Contractor Tendering

Automated tender, structured quotes, digital instruction — no phone calls

E.1 How procurement works in Propra

  • Work order raised by owner (or flagged by compliance tracker): description, property, urgency, estimated value

  • Propra AI generates scope of works document from the description — structured, with standard clauses, H&S requirements, access instructions

  • Scope sent to vetted contractor panel via API — panel organised by trade, geography, Propra rating

  • Contractors submit quotes in Propra standard format (not PDF emails) — structured data: total, breakdown, start date, duration, warranty

  • Owner sees quotes side-by-side on screen — not as separate PDF emails to interpret manually

  • Owner selects: instruction issued automatically via API. Contractor receives formal instruction with scope and commercial terms.

  • Job tracked: contractor updates status (mobilised, in progress, snagging, complete). Photos uploaded via contractor app.

  • Completion: owner approves on screen. Payment instruction generated. Propra earns 3% of contract value.



E.2 Contractor panel — data requirements

  • Every panel contractor: Companies House verification (API), PLI and employers liability insurance verified (API), RAMS templates on file, previous job ratings in Propra

  • Propra does not add contractors who are not API-verified — no manual onboarding of contractors by an admin

  • New contractor applications: self-serve form, automated Companies House check, insurance certificate OCR extraction, approval workflow

H Transaction Room

Full digital deal management — sale, purchase, or refinancing — from heads of terms to completion

Purpose

A commercial property transaction currently involves 15+ parties exchanging information via email over 3-6 months. Propra brings all parties into one digital room: owner, solicitor, agent, buyer, lender. Every document, every status update, every approval — in one place. Propra tracks it. Propra chases it. Nothing falls through the cracks.



H.1 Transaction types supported

  • Sale of investment property: from instruction to agent/buyer through to completion

  • Purchase of new asset: from identification through due diligence to completion

  • Refinancing: from lender selection through legal charge to drawdown

  • Lease renewal / regear: from heads of terms through to new lease execution



H.2 Transaction Room components

  • Deal overview: parties, asset, deal type, key dates, status

  • Document vault: all transaction documents versioned and party-accessible (NDA-gated for buyers)

  • Data room: due diligence pack auto-assembled from Propra data (title, leases, EPC, planning, searches, insurance, financial summary)

  • Milestone tracker: offer received, heads agreed, solicitors instructed, searches ordered, searches received, contract issued, exchange, completion — live status from API feeds where available

  • NDA workflow: buyer receives NDA link, signs digitally (DocuSign or Docusign alternative API), access granted automatically

  • Offer management: multiple offer tracking, accept/decline, counter-offer, best and final

  • Communication log: all party communications logged — not replacing email but capturing outputs in one place



H.3 Integration requirements

  • DocuSign API or HelloSign/Dropbox Sign API — digital signature for NDAs, contracts, legal documents

  • SearchFlow / TM Group / InfoTrack — search ordering from within Transaction Room

  • HMLR API — land registry updates, AP1/TR1 form generation

  • Lender portal APIs (where available) — for refinance deal tracking

  • Claude API — due diligence query answering from data room documents

K Valuation & AVM

Automated property valuation on every asset — updated monthly

K.1 How automated valuation works in Propra

  • On property add: AVM runs immediately using ATTOM (US) or CoStar AVM (UK/US) — returns estimated market value, confidence score, comparable sales used

  • Monthly refresh: AVM runs automatically on all portfolio assets — updated value appears on dashboard

  • Valuation history: chart showing how each property value has moved over time

  • Yield calculation: auto-calculated from AVM value + NOI from accounting sync

  • Valuation report: branded PDF export using AVM data + CoStar market commentary + Propra portfolio context — replaces £1k-£3k formal desktop valuation for portfolio monitoring purposes

  • Caveat shown clearly: AVM is for portfolio monitoring. Formal RICS valuation required for lending, legal, or regulatory purposes — Propra can instruct RICS valuer via panel API



K.2 Integration requirements

  • ATTOM AVM API — US commercial property automated valuation

  • CoStar AVM / Analytics API — US and UK commercial AVM

  • CoreLogic Property Data API — US valuation and risk data

  • CBRE Valuation API (where available via data partnership) — institutional grade AVM reference

  • RICS registered valuer panel — for formal instructions (API instruction, same as surveyor panel)
