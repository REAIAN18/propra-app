1. Purpose of This Document

This is the single source of truth.

It supersedes all previous briefs, demos, and verbal instructions. Every screen, feature, API integration, data source, design decision, and UX pattern is defined here. The team builds exactly what is in this document — no interpretation required.

What failed before — and why this document exists differently

The original brief was excellent but was given to a builder who extracted the data layer and built a display product. Propra is not a dashboard. Propra is an execution engine. Every screen must answer three questions:

  • What is the issue and how much is it costing?

  • What exactly will Propra do about it?

  • What does the owner need to tap to make it happen?

The demo artifact (PropIQ) captured this philosophy correctly. The built version did not. This document rebuilds the specification from first principles, incorporating the original brief, the demo design language, the updated visual direction, and all real-world API integrations.

3.3 Layout System

  • Sidebar: 220px fixed, dark navy (#0B1622), grouped navigation with section labels and badge counts

  • Top bar: 64px, white, page title left + search + action buttons right (Export / + Add Property / Run Full Analysis)

  • Alert banner: Full width below top bar, amber background for warnings, actionable link right-aligned

  • Content area: White cards on #F7F8FA background, 24px padding, 16px gap between cards

  • Card borders: 2px left border colour-coded (green = good, amber = review, red = urgent)

  • Hero section: Dark navy card at top of dashboard with portfolio greeting, date, Portfolio Value Score widget

  • Stat row: 7–8 KPI tiles in a scrollable horizontal row below hero

  • Responsive: Desktop-first (1280px+), tablet breakpoint (768px), mobile (390px)

3.4 Component Patterns

  • Opportunity card: Category badge (RENT UPLIFT / INSURANCE / REFINANCE etc.) + large £$X,XXX/yr figure + plain-English description + evidence sentence + Quick win badge (if <4 weeks) + primary CTA

  • Action item: Icon + title + property name + days remaining + View [section] → link

  • KPI tile: Label (small caps) + large number (Instrument Serif) + trend arrow + benchmark note

  • Progress bar: Labelled horizontal bar, colour shows status, % and absolute value

  • Portfolio score widget: Circular gauge 0–100, sub-scores (Income / Cost / Growth), status label

  • Alert banner: Warning icon + bold label + plain sentence + "Review now →" right-aligned

  • Table row: Property name + address + metrics + status badge + action button inline

5.1 Value Dashboard (/dashboard)

Purpose

The first screen every user sees. Must immediately show: portfolio health score, total value at risk, unactioned opportunity £$, and ranked list of actions. A user should be able to understand the state of their entire portfolio and know exactly what to do next within 10 seconds of loading.

Layout (top to bottom):

  • Alert Banner — conditional, amber if any urgent compliance/insurance items. Shows item count + estimated fine/risk exposure + "Review now →"

  • Hero Card (dark navy) — "Good morning [Name] — here's your portfolio" + date + portfolio subtitle + Portfolio Value Score widget (circular gauge 0–100 with sub-scores)

  • KPI Strip — 7 tiles: Portfolio Value | Gross Monthly Rent | NOI | Occupancy % | Total Sq Ft | Avg NOI Yield | Costs Saved YTD + Unactioned Opportunity tile (amber, always last)

  • 3-column analytics row — Geographic Spread (horizontal bar per market) | Asset Class Mix (dot chart) | Top 5 by NOI Yield (ranked list)

  • AI Opportunity Centre — heading "Ranked by annual value — every action is executable inside Propra" + 3-column card grid. First card highlighted as HIGHEST ROI. Each card: category badge + £$X,XXX/yr headline + plain-English description + evidence sentence + time estimate + CTA button

  • Lease Expiry Profile — stacked bar chart by quarter (Q1–Q8) with urgent/review/secure colour coding

  • AI Opportunity Summary — 4 tiles: Income | Cost Saves | Refi/Value | Value Uplift — each with £$ total and action count

  • Portfolio Assets table — all properties with G2N, Occ%, ERV uplift badge, quick actions

  • Action Required — right column feed of urgent items with property, days, and "View [section] →"

KPI Data Sources:

  • Portfolio Value: Sum of per-property estimated values (CoStar AVM or manual override)

  • GMR/NOI: Aggregated from lease data + Xero/QuickBooks accounting integration

  • Occupancy: Active leases / total lettable area — live from lease register

  • NOI Yield: NOI / Portfolio Value — calculated field

  • Costs Saved: Running total of Propra-executed savings (insurance + energy + CAM)

  • Unactioned Opportunity: Sum of all open AI-identified opportunities not yet actioned

5.2 Insurance Audit (/insurance)

Purpose

Show exactly how much the portfolio is overpaying on insurance vs live market benchmarks, then place replacement policies direct with carriers — no broker. Propra earns 15% of year-1 saving.

  • Per-policy table: Policy type | Property/ies | Current annual premium | AI market rate | Saving | Status badge (Overpaying / Competitive / Negligible)

  • Total row: "Total identified saving: £$XX,XXX/yr across [N] policies" + "Get quotes inside Propra →"

  • Placement workflow (modal/drawer): Carrier comparison grid with live quotes, terms, excess — select + confirm to place

  • Placed policies section: Active placements showing carrier, policy number, next renewal, Propra commission earned

5.4 Refinance Centre (/refinance) ← NEW SCREEN

Purpose

Propra monitors every loan in the portfolio daily against live SOFR, LIBOR, and lender rates. When a refinance opportunity exists — rate above market, DSCR improving, LTV headroom available — Propra surfaces it with the annual saving, pre-vetted lender options, and a one-tap "Explore lenders →" workflow.

Layout:

  • Header: "Refinance Centre — AI monitors your loans daily vs live SOFR/market rates" + "Compare lenders →"

  • Loan table: Property | Current rate | Market rate | Balance | DSCR | Annual saving at refi | Action

  • Status logic: Current rate > market rate + 25bps AND DSCR > 1.25 → "Refi now" (green). Within 25bps → "Monitor" (gray). Below DSCR threshold → "Hold" (amber)

  • Lender comparison drawer: When "Refi now" clicked — shows pre-vetted lenders, indicative rates, fees, LTV, DSCR requirements. "Request quote" sends Propra-brokered inquiry

  • DSCR tracker: Per-property NOI / Debt service — live chart with trend

  • LTV calculator: Current valuation / outstanding debt — flags when headroom opens

  • Rate alert system: User sets threshold (e.g. "alert me if market drops below 5%") — push notification + email

  • Completed refinances: Track closed deals, rate achieved, Propra savings total, commission earned

5.5 Acquisitions Scout (/acquisitions)

Purpose

AI continuously screens live market listings against the portfolio's acquisition criteria (type, geography, cap rate, deal size, asset class). Scores each deal for fit, identifies income optimisation potential, and surfaces it in a ranked pipeline. Propra earns 0.5–1% on completed acquisitions.

Layout:

  • Criteria panel: Portfolio type preference, geography, target cap rate range, deal size, asset class weights — editable

  • Live pipeline: Ranked deals with AI fit score, address, type, asking price, cap rate, NOI, estimated income uplift, days on market, source

  • Deal card expanded: Full AI underwriting summary, comparable sales, income opportunity breakdown, "Request details" / "Track deal" / "Pass" actions

  • Closed deals tracker: Acquisition pipeline with offer, due diligence, exchange, completion status

  • Market intel panel: CoStar market rent trends, cap rate movement, transaction volume by sector

Real-world integrations required:

  • Crexi API — commercial real estate listings and analytics

  • RealPage / CoStar Market Analytics — rent benchmarks, vacancy, absorption

  • EG Group / EG PropertyLink API (UK) — UK investment and occupier market data

  • Auction.com API — distressed commercial property pipeline

  • ATTOM Data Solutions — ownership records, debt, transaction history

5.6 Hold vs Sell (/hold-sell)

Purpose

Per-asset scenario modelling: Sell Now vs Hold 3yr vs Reposition. Shows IRR, net proceeds, estimated exit value, income gap. Replaces £20k–£50k of advisory fees per decision. AI provides a clear recommendation with rationale.

Layout:

  • Asset selector: Choose property or view all as cards

  • Scenario matrix: 3 columns (Sell Now / Hold 3yr / Reposition) × rows (Gross proceeds, Costs, Net proceeds, IRR, Annual yield, Risk rating)

  • AI recommendation: Dark card with green/amber/red recommendation label + plain-English rationale + "See full analysis"

  • Assumptions panel: Toggle entry price, exit yield assumption, CapEx estimate, rent growth assumption — scenario updates live

  • Comparable exit data: Recent sales of similar assets in same market with achieved cap rates

  • Tax impact calculator: CGT / SDLT / depreciation recapture estimate per scenario

Real-world integrations required:

  • CoStar Capital Markets — comparable sales, cap rate benchmarks, exit yield assumptions

  • MSCI Real Capital Analytics — transaction benchmarks by asset class and geography

  • ATTOM — AVM valuations, recent sales, ownership history

  • IRS/HMRC tax data — CGT rates, SDLT/transfer tax tables (static, updated annually)

  • Anthropic Claude API — scenario narrative, recommendation rationale, risk factors

5.7 Compliance (/compliance)

Purpose

Every certificate (EPC, fire safety, legionella, asbestos, contractor insurance, EICR) tracked with expiry countdown. Fine exposure shown in £/$. One-tap certificate ordering from vetted surveyors. Prevents insurance void risk and regulatory fines averaging £8k–£15k per incident.

Layout:

  • Summary: Total fine exposure (red number) + certificates expiring <30 days + expired count

  • Certificate table: Property | Certificate type | Status | Expiry date | Days remaining | Fine exposure | Action

  • Status colours: Green = valid >90d | Amber = expiring <90d | Red = expired or <30d

  • Per-item action: "Order now" links to vetted surveyor panel with prices + availability

  • Document vault: PDF upload for existing certificates, OCR extraction of expiry date and property

  • Reminder system: Auto-email at 90 days, 30 days, 7 days before expiry

Real-world integrations required:

  • MHCLG / VOA (UK) — EPC register lookup by address (public API)

  • Elmhurst / Stroma Certification — EPC ordering marketplace

  • Surveyors network API — RICS-accredited surveyor panel for asbestos, legionella, fire risk, EICR

  • Companies House API — contractor insurance verification

  • Anthropic Claude API + PDF parsing — OCR and extract expiry dates from uploaded certificates

  • AWS Textract or Google Document AI — certificate PDF extraction fallback

5.8 Rent Clock (/rent-clock)

Purpose

Every lease event tracked with a live countdown. Tenant intelligence scores (willingness to pay, move cost, leverage). Propra auto-drafts renewal letters and strategy notes. Prevents rent drift — avg 14% below-ERV gap closed at renewal.

Layout:

  • Urgent events: Top section, red cards — lease expiries < 90 days, breaks exercisable now

  • Review soon: Amber cards — lease reviews due in 90–365 days

  • Secure: Green cards — leases with >1 year until next event

  • Each card: Tenant name + property + event type + countdown clock + current rent + ERV + gap % + leverage score + "Draft letter" / "Engage tenant" CTAs

  • Leverage score: AI-generated 0–10 score based on tenant sector health, relocation cost, market availability — with explanation

  • Renewal strategy: AI-recommended approach (push for open market / offer incentive / fix and extend) with rationale

Real-world integrations required:

  • CoStar Tenant Intelligence — tenant business health, sector data, relocation activity

  • Companies House API — tenant financial health, credit, director changes

  • D&B Hoovers / Creditsafe API — commercial tenant credit scoring

  • CoStar / EGi rental comparables — ERV validation by asset type and location

  • Anthropic Claude API — leverage score narrative, renewal strategy, draft letter generation

5.11 Ask Propra AI (/ask)

Purpose

Full-context AI assistant with the entire portfolio pre-loaded. Every screen also has a context-aware AI panel with pre-built prompts specific to that screen. Not a chatbot bolted on — AI is the backbone of the product.

Technical requirements:

  • Anthropic Claude API (claude-sonnet-4-20250514) — primary AI model

  • Context window must include: all property data, lease register, insurance schedules, energy data, compliance status, open opportunities, recent AI actions

  • Per-screen prompt library: Each screen has 4-6 pre-written prompts per screen context. Insurance examples: biggest overpay, draft insurer letter, hurricane exposure explanation

  • Free-form: User can ask anything in plain English — "Which tenant is most likely to leave?", "What would happen to my NOI if I refinanced Brickell?"

  • Action execution from chat: AI can trigger actions directly. Example: Switch the Wynwood energy contract executes via energy API

  • PDF generation: Any AI response can be exported as a branded PDF report

  • Memory: Conversation history stored per user/portfolio — AI remembers context across sessions

12. Acceptance Criteria

How to know when each feature is done — not just built

Definition of done

A feature is not done when the code works in isolation. It is done when: (1) it shows real data from the correct source, (2) the action it enables actually executes end-to-end, (3) the output matches the design spec exactly, and (4) it has been tested on a real portfolio with real documents.

Insurance placement — done when:

  • Upload a real insurance schedule PDF → system extracts all policy details correctly

  • System benchmarks against live carrier APIs and shows real saving (not mocked data)

  • User can select a replacement policy, confirm, and a real placement instruction is sent to the carrier

  • Commission is recorded in the Propra ledger on placement confirmation

Refinance Centre — done when:

  • Daily SOFR rate is fetched from FRED API and reflected in the rate comparison table

  • Loan data entered manually or extracted from uploaded document shows correct DSCR

  • Clicking "Explore lenders →" shows real lender panel with indicative rates

  • Rate alert emails are sent when threshold is crossed

Dashboard — done when:

  • All KPI tiles show real data from lease register + accounting integration, not demo placeholders

  • AI Opportunity Centre shows real identified opportunities ranked by annual value

  • Portfolio Value Score reflects actual portfolio metrics, not a hardcoded number

  • A new user can log in, upload their lease data and a utility bill, and within 5 minutes see real personalised intelligence on their portfolio

Ask Propra AI — done when:

  • AI has access to the user's real portfolio data in its context window

  • "What is my biggest insurance overpay?" returns the correct answer from real data

  • "Draft a renewal letter for [tenant name]" produces a complete, accurate, usable letter

  • AI can trigger an action ("switch the Wynwood energy contract") and the switch executes
