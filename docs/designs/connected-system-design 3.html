Core Architecture · v1

One Property. One Living Profile. Every Purpose.

RealHQ is NOT 16 modules. It's one intelligent property profile that adapts its face for different audiences. The owner builds the profile through normal use. Then one click generates whatever they need: lender pack, insurance submission, marketing brochure, letting particulars, valuation brief, investment memo. Same data. Different packaging.

The Property Profile

The Central Entity

Every property starts thin — just an address. Every interaction makes it richer: auto-fetched public data, uploaded documents, Claude-extracted lease terms, insurance quotes, energy analysis, compliance certs, tenant data, comparable evidence, planning potential, financial modelling. That profile IS the product.

Once you have a rich, verified, always-current profile, you generate ANY output from it:

Need acquisition finance? → lender pack from the profile. One click.
Need insurance quotes? → insurance submission from the profile. One click.
Want to sell? → marketing brochure from the profile. One click.
Want to refinance? → lender information memorandum. One click.
Need to instruct a solicitor? → instruction letter with all property details. One click.
Want to attract a tenant? → letting particulars brochure. One click.
Need a valuation? → valuation briefing pack. One click.

The Portal: One Link, Many Viewers

Every property has a unique portal URL: realhq.com/p/[propertyId]. What you see depends on who you are.

OWNER VIEW (full access)
Everything. Every module, every document, every number, every action. The 16-module dashboard experience.

LENDER VIEW (shared for financing)
Property overview, financial summary, NOI, occupancy, WAULT, rent roll, tenant schedule with covenant ratings, comparable evidence, insurance details, compliance status, shared documents. NOT shown: owner's Hold vs Sell analysis, refinance modelling, personal notes.

INSURER VIEW (shared for quoting)
Property type, construction, age, sqft, rebuild value, flood zone, fire cert status, claims history, occupancy, current coverage. Auto-submitted to CoverForce API AND/OR opened as a portal for a broker. NOT shown: financial performance, rent reviews, commercially sensitive data.

BUYER VIEW (shared for marketing/disposal)
Beautiful property presentation, investment highlights, tenant schedule, location analysis, comparables, planning potential, data room (NDA-gated). PLUS the "Opportunity Page" — what a buyer can unlock with RealHQ on day 1. NOT shown: what the owner paid, their financing, internal analysis.

TENANT VIEW (shared for engagement)
Building info, shared amenities, management contacts, their lease summary, planned works, communication channel with landlord. NOT shown: other tenants' rents, financial performance, owner strategy.
The unlock

The owner builds the profile through normal use. They never re-enter data. They never recreate documents. They never manually build lender packs or marketing brochures. RealHQ generates everything from what it already knows.

Flow 1: Buy → Finance → Insure

Scout finds a deal → deal room opens → analysis, documents, and comparables build the property profile automatically.

User needs acquisition finance → one click: "Get financing" → RealHQ assembles a lender pack from the deal profile. PDF + interactive portal + Excel model. Send to 3 lenders simultaneously. View tracking shows who opened it, what they looked at, how long. Compare term sheets when they come back. Selected terms feed into the financial model automatically.

Insurance on completion day → property profile already has what insurers need (type, construction, size, rebuild value, flood zone). One click → CoverForce returns quotes in seconds. Place policy → dashboard updates → compliance module confirms coverage is adequate → refinance centre confirms lender covenants met.

Flow 2: Manage → Optimise

Every module runs against the property profile automatically:

Energy: EPA/EIA benchmarks before any upload. Google Solar for roof potential. Upload bills → Claude extracts → exact analysis against comparables. Market-context-aware recommendations (regulated vs deregulated).

Insurance: Annual re-quote 60 days before renewal. Auto-compare against current terms. Alert if better rates available.

Rent: Review dates monitored. Comparable evidence from CoStar. Letters drafted by Claude. Sent via Resend. Outcomes tracked.

Compliance: Requirements auto-detected from market context. Cert expiry monitored. Fine exposure calculated. Procurement links for expired certs.

Dashboard: Aggregation of all property profiles. KPIs, alerts, top opportunities ranked by value. "Since using RealHQ: X actions taken, $Y total value found."

Flow 3: Sell (or Refinance)

Hold vs Sell shows three scenarios: sell now, hold, or optimise-then-sell. All modelled with real data from the (now much richer) property profile.

If sell: One click → marketing brochure + public portal + NDA-gated data room. All auto-generated from the profile. Zero new document preparation. The brochure includes the "Opportunity Page" showing buyers what RealHQ can unlock — making RealHQ-marketed properties more attractive than traditional listings.

If refinance: Same lender pack template, same portal system. "Refinance" instead of "acquisition" in the request section. One property profile → one click → lender-ready pack for any purpose.

If hold: Continue optimising. The profile keeps getting richer. Every saving compounds.

Flow 4: Repeat

Capital from sale → Scout is already looking for the next deal matching the updated portfolio profile. "You sold your Coral Springs industrial. Here are 5 replacement properties that maintain your yield and add diversification." Every step uses and enriches the same profile. By the time you sell, that profile is the most comprehensive property dossier in existence — and the disposal materials wrote themselves.

The Document Generator

One engine, many templates

Property profile + audience template + Claude narrative → PDF + portal + Excel. Built once, used for: Investment Memo, Lender Pack, Insurance Submission, Marketing Brochure, Letting Particulars, Valuation Brief, Rent Review Letter, LOI/Heads of Terms, Solicitor Instruction, NDA, CAM Reconciliation, Planning Statement, Accountant Tax Export.

What varies per template: Which fields to include. What to redact. Tone (lender = risk-focused, buyer = opportunity-focused, tenant = welcoming). Format (PDF, portal, Excel). Whether to include Claude narrative or just data.

The Portal System

Permissions + view tracking + NDA gating

Every shared link has: recipient email, permission level (which sections visible), NDA requirement (DocuSign before access), view tracking (opened, pages viewed, duration). The owner sees a CRM-like dashboard: "Bank A spent 4 minutes on financials. Broker B hasn't opened the submission. Buyer C signed the NDA and is browsing the data room."

The "Opportunity Page" — Why This Changes Marketing

No other platform includes this in a disposal brochure. Traditional brochures show: here's the rent, here's the yield. RealHQ's brochure adds: "Here's what a buyer can unlock on day 1: $6k insurance saving ($86k value), $4k energy saving ($57k value), rent review potential of $54k/yr ($771k value). Effective yield if optimisations executed: 9.3% vs headline 7.8%."

This makes every RealHQ-marketed property more attractive. The buyer sees immediate upside. That pulls buyers onto the platform too — the acquisition funnel opens from both sides.

What This Means for the Build

5 systems, not 16 pages

1. Property Profile — central data model. Everything reads/writes here.
2. Document Generator — one engine, many audience templates → PDF + portal + Excel.
3. Portal System — permissions + view tracking + NDA gating + commenting.
4. Action Engine — Resend, CoverForce, DocuSign, EnergySage, Google APIs. Shared primitives.
5. Learning Loop — every interaction enriches the profile and improves recommendations.

The "modules" are views — configurations of which profile data to show, which template to use, which action to call. Build the systems. The views are configuration on top. A new module goes from weeks to days because the infrastructure exists.

Why the connections are the moat

Anyone can build an insurance comparison. Anyone can build a rent tracker. Nobody else has the combined intelligence where an insurance saving automatically improves Hold vs Sell, where a Scout acquisition auto-triggers insurance + energy + compliance, where marketing a property requires zero new data because the profile built itself over years of normal use. The connections are the product. The modules are just windows into it.
