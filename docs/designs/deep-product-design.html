Insurance Audit /insurance

Property profile → instant benchmarks → upload schedule → CoverForce instant quotes → place → track → annual renewal cycle. Frictionless: the profile IS the insurance submission.

Before upload — instant from profile

Property type + sqft + location + flood zone + building age → estimated premium range from BOMA benchmarks + FEMA/EA risk data. "Properties like yours typically pay $8–$14/sqft. Upload your schedule for exact comparison." This appears the moment a property exists — no action needed.

Upload → extract → compare (30 seconds)

Claude extracts: carrier, premium per property, coverage types, rebuild values, excess, policy dates. Compare against benchmarks → "You're paying $12/sqft. Market mid: $9.50. Potential saving: $X/yr." Every number tagged estimated or verified.

Get quotes — instant via CoverForce API

One click. Property profile already has what CoverForce needs (the profile IS the submission). Multiple carrier quotes return in seconds. Comparison table: carrier, premium, coverage, excess, rating. "Best quote saves $18k/yr." Fallback: Ivans API, or generate a broker submission portal from the Document Generator. Broker gets a beautiful portal link with view tracking.

Place — one click

"Accept this quote" → confirmation: old vs new policy, saving amount, start date. CoverForce API binds policy. New policy details flow back into the property profile. Dashboard updates. Compliance module confirms coverage adequate. Refinance Centre confirms lender covenants met.

Ongoing — annual auto-cycle

60 days before renewal → auto-re-quote → alert: "3 quotes below your current rate." The user approves, RealHQ executes. The cycle repeats without effort. Every placement outcome (actual saving vs estimate) feeds back into benchmark accuracy.

Every quote, every placement, every actual saving vs estimate → improves benchmarks for the next similar property. "Across our platform, 50,000 sqft offices in FL save an average of 24%. CoverForce Carrier X wins 60% of comparable placements."
Insurance is also a portal output

The same property profile → "Generate insurance submission" → Document Generator creates a branded submission pack → share via portal link to a broker if CoverForce doesn't cover the risk. View tracking shows when the broker opened it. This is the connected system in action — not a separate insurance product, but a view of the property profile with an action (get quotes) and a document template (insurance submission).

Actions — market-dependent

Deregulated markets (TX, UK, etc.)

EnergySage / Power Kiosk API → submit usage profile → receive supplier quotes → user selects → switch initiated. Plus all regulated-market options below.

Regulated markets (FL, etc.)

Every bill uploaded, every tariff switch, every solar PPA installed → improves the benchmark for the next similar property. "Buildings like yours that switched to demand-response tariff saved an average of 12%."

Financial appraisal

Detailed development model: land value, construction cost (BCIS UK / RSMeans US), professional fees (12-15%), contingency, finance costs, marketing, GDV, profit. All interactive — user adjusts inputs, model recalculates. Export as branded PDF + Excel.

Comparable evidence pack

Auto-generated: nearby approved schemes (planning data), sold comparable developments (Land Registry/CoStar), market values for proposed end use. The document a planning consultant would produce — RealHQ generates the first draft.

Planning statement

Claude drafts from template: site context, proposal, policy compliance (referencing specific local plan policies by name and number), design rationale, precedent. The core document for any application.

Pre-application letter

Claude drafts the pre-app enquiry: site description, proposal, key questions. UK: most authorities offer pre-app. US: identifies relevant municipal planning department and correct form.

Application document checklist

□ Planning Statement — auto-drafted by Claude, review needed
□ Design & Access Statement — outline drafted, needs architect input
□ Floor Plans / Elevations — user uploads from architect
□ Flood Risk Assessment — flagged: property in flood zone, link to specialists
□ Heritage Statement — not required (not listed/conservation)
□ Transport Assessment — basic assessment from Walk Score + traffic data
Each: auto-drafted where possible, flagged where specialist needed, with links to find consultants.

Post-decision

Approved → development opportunity feeds into Hold vs Sell: "Property now worth $X more with planning consent. Sell with consent or develop?" Refused → Claude analyses refusal reasons → suggests: appeal, revised application, or move on.

Every planning analysis, every user response, every real-world outcome → improves identification of realistic opportunities. "Class MA conversions in [borough] have 85% approval rate."

Hold vs Sell /hold-sell

Not a DCF calculator. Three scenarios modelled with real data from the property profile. The critical third option: optimise then sell — where every other module's value converges.

Three scenarios, auto-modelled from profile data

Scenario A: Sell Now

Market value (from CoStar comps or user's valuation) → selling costs (agent 1-2%, legal, CGT/capital gains — UK and US specific) → outstanding debt → net proceeds. "If you sell today: net $X. Reinvested at current yields: $Y/yr income."

Scenario B: Hold and Do Nothing

Year-by-year cashflow: rental income (from actual lease data in profile), opex, NOI, debt service, free cashflow. Exit year: NOI ÷ exit cap rate – costs – debt. 3/5/10yr projections. IRR, equity multiple, total return.

Scenario C: Optimise Then Sell (the RealHQ play)

Model executing EVERY optimisation RealHQ has identified: - Insurance saving: $18k/yr → at 7% cap = $257k value created - Energy saving: $12k/yr → $171k value - Rent review uplift: $54k/yr → $771k value - WAULT improvement (lease restructuring): cap rate compression → value impact - Planning uplift: if development potential exists → value with consent "Optimise for 18 months then sell: $1.2M more than selling today. IRR on optimisation effort: 22%. Largest driver: rent review ($771k) — costs only a letter to initiate."

Side-by-side: [Sell Now] | [Hold 5yr] | [Optimise 18mo + Sell]
Each: net proceeds, IRR, equity multiple, annual cashflow.
Sensitivity toggle: cap rates +/- 50bps, growth 2% vs 3%.
All interactive — drag sliders, numbers update in real time.
Claude recommendation highlighted. Shareable as PDF + portal link.
When user eventually sells or holds, record decision + actual outcome. "Users who chose Scenario C achieved 15% higher returns on average."

Refinance Centre /financing

Continuous debt monitoring, not a one-time calculator. Models scenarios, monitors maturity, generates lender packs from the Document Generator.

Current debt dashboard (from profile)

User enters loan details (or uploads facility letter for Claude extraction). Shows: outstanding balance, rate, maturity, LTV, DSCR, ICR, covenant headroom. Market comparison: "Your rate: 6.5%. Current market: 5.6-6.0% (FRED SOFR + Chatham spread data). Saving: $14.7k/yr."

Three refinance scenarios

Rate reduction: Same amount, better rate. Show annual saving, breakage cost, net benefit, payback period.
Cash-out: Property appreciated → higher LTV → cash released. Show: new debt service, DSCR (still compliant?), cash released, what to reinvest in (link to Scout).
Restructure: Variable→fixed (or reverse), extend term, switch to interest-only. Full impact on cashflow + total cost.

Lender pack — one click from Document Generator

Property profile + lender template → branded PDF + portal + Excel model. Send to multiple lenders. View tracking: who opened, which pages, how long. Compare term sheets when they respond.

Maturity monitoring

12 months before maturity → amber alert. 6 months → urgent. Auto-generate refinance request document. "Your loan matures in 12 months. Current market rates are X%. Start exploring."

CAM Recovery /work-orders

Finding money already spent but not recovered from tenants.

Flow

Step 1: Claude extracts from leases: each tenant's CAM share %, cap, exclusions, reconciliation date.
Step 2: User enters or uploads actual CAM expenses by category (maintenance, cleaning, security, insurance, utilities).
Step 3: Gap analysis: "Spent $120k. Recoverable: $108k. Billed: $95k. Under-recovery: $13k." Per-tenant breakdown showing who's under-billed and by how much.
Step 4: Generate year-end CAM reconciliation statement per tenant — branded PDF via Document Generator. Send via Resend or download. Flag: "Tenant C's cap is below costs. At renewal, negotiate removal."

Lease Restructuring /lease-restructuring

Modelling "what if" on every lease. A restructure can create more value than a rent review — extending from 3yr to 10yr can compress cap rate by 100bps+ = $500k+ on a $5M property.

Scenario modelling per tenant

Current: 3yr unexpired, $25/sqft, break at year 2 Scenario A: Extend to 10yr Tenant gets: certainty, fit-out contribution Landlord gets: +7yr WAULT → cap rate compression 7.5%→6.5% → value uplift $X Offer: 3 months rent-free for 10-year commitment Scenario B: Regear with rental uplift Extend 7yr at $28/sqft + 6mo rent-free Tenant gets: $70k saving, certainty Landlord gets: $36k/yr uplift ongoing + 4yr WAULT improvement NPV: +$180k to landlord Scenario C: Surrender and re-let Accept surrender fee $X. Re-let at market $29/sqft. Void risk: 3-6 months. Fit-out: $Y. NPV depends on void. Each: full cashflow model, NPV comparison, risk flags.

Heads of Terms: Claude generates from template + scenario specifics. Parties, property, current terms, proposed terms, conditions. Branded PDF. Send via Resend or DocuSign. Track: Proposed → Negotiating → Agreed → Lease Completed. WAULT and portfolio value update on completion.

Compliance /compliance

Auto-detect requirements from market context. Risk quantification with actual fine amounts. Procurement connections. Certificate upload with AI extraction.

Auto-detection

Market context + property type + building age → full compliance requirement list with legislation reference, frequency, and fine amount. UK: EPC, Fire Risk Assessment, Asbestos Survey, Legionella, Gas Safety, EICR, MEES. US: ADA, OSHA, state fire codes, elevator cert. FL-specific: hurricane shutters, wind mitigation.

Risk quantification

Each expired cert → specific fine from legislation. "6 violations, £116k maximum fine exposure. 2 items prevent legal letting (EPC below minimum, no gas safety cert). These aren't just fines — they're void leases."

Procurement + lifecycle

"Get a quote" → connects to service provider networks. Upload completed cert → Claude extracts: type, date, expiry, outcome, assessor. Auto-updates dashboard. Next expiry alert auto-set. Insurance implications flagged: "Expired fire risk assessment may void your buildings insurance fire cover."

Tax /tax

Tax awareness, not advice. Market-specific calculators. Accountant export.

UK

SDLT calculator (interactive — change price, see tax). CGT estimator (purchase price, current value, holding period, ownership structure). ATED check (company-held, >£500k). Business rates from VOA API — flag if rating appeal likely.

US

Property tax from county assessor data — flag if assessed value seems high vs market (appeal opportunity). Depreciation schedule (39yr straight-line). 1031 exchange eligibility: "Defer $X of gains by reinvesting within 180 days. See Scout for replacement properties." Cost segregation flag: "Typical first-year benefit: $X."

Both

Every screen: "Estimates for planning purposes. Consult your tax advisor." One-click "Export for accountant" → branded PDF via Document Generator with all property-level tax data. Saves the accountant hours of data gathering.

The Learning System (Cross-Module)

How RealHQ gets smarter the more it's used. The flywheel.

What gets stored

Property level: Every document uploaded builds the data profile. Every analysis refines benchmarks. Every action outcome (actual saving vs estimate) improves predictions.

User level: Preference profile (Scout interactions), risk appetite (decision patterns), engagement patterns (which modules used/ignored).

Platform level: Aggregated anonymised data across all users. "Average insurance saving: 22%. Average rent review uplift: 12%." These improve every individual analysis.

The flywheel

More users → more data → better benchmarks → better recommendations → more actions taken → more outcomes recorded → better predictions → more value → more users.

How it manifests over time

Month 1: "Based on market data, your insurance is likely 20-30% above market."
Month 6 (50 users): "Based on 50 portfolios, offices your size typically save 24%."
Month 12 (200 users): "Properties identical to yours saved avg $18.4k. Best carrier for your profile: [X] — won 60% of comparable placements at best rate."

The product gets more valuable with every user, every document, every outcome. That's the moat.

The product isn't 16 modules

It's one intelligence layer that understands commercial property, gets smarter with use, and executes on what it finds. The modules are views — different angles on the same understanding. Every module feeds every other module. An insurance saving improves NOI → changes Hold vs Sell → affects refinance options → impacts acquisition strategy. It's one system with 16 windows. The agents need to build it that way.
