First Load: Zero Questions, Instant Results

The principle

The user's portfolio IS the acquisition criteria. RealHQ already knows: what they own, where they invest, what yields, what types, what tenant profiles. Don't ask — infer. Show results immediately. Let the user refine by reacting, not by filling forms.

Every metric defined

Cap Rate: NOI ÷ Price. The key number. Green if above portfolio avg.
WAULT: Weighted Average Unexpired Lease Term. Longer = more secure.
Est. NOI: Rent – opex (BOMA benchmark). Flagged as estimate until verified.
DSCR: NOI ÷ debt service (65% LTV, current SOFR + spread from FRED). Above 1.25x = green.
5yr IRR: Purchase → NOI growth → exit at market cap rate – costs. Total return.
Comparables: Recent sales nearby (CoStar/Land Registry). Is this deal above, at, or below market?
Day 1 Value-Add: What RealHQ estimates it can find from day one. Saving ÷ cap rate = value created.

Match score algorithm (0–100%)

30% yield vs portfolio · 20% location fit · 15% property type · 15% risk profile match · 10% deal size · 10% value-add potential. Weights adjust as the system learns from user reactions.

The Pattern Every Module Should Follow

7 principles from Scout

1. Instant value from existing data — don't ask, infer. Show confidence level.
2. Refine through reactions, not forms — one-tap feedback, not multi-field forms.
3. Every number tagged verified or estimated with source.
4. Every analysis generates a shareable document — PDF + portal link.
5. Every action has a full lifecycle — data → recommendation → document → sign → send → track → outcome.
6. Every outcome feeds back — actual vs projected improves future estimates.
7. Everything connects — Scout acquisition feeds Hold vs Sell feeds Refinance feeds the next Scout search.
